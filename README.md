# VSVH Languages — умная платформа для изучения языков

Современная онлайн-платформа, объединяющая роли студента и преподавателя. Демонстрационный прототип включает каталог курсов, фильтрацию и сортировку, отслеживание прогресса, автопроверку упражнений, генерацию отчётов (PDF/CSV) и сохранение настроек в localStorage.

## 🚀 Основные возможности

- **Ролевая модель** – студент, преподаватель, администратор (переключение ролей в демо).
- **Каталог курсов** – фильтрация по языку, уровню (A1–C2), рейтингу; сортировка по популярности, дате, оценке.
- **Управление студентами** – таблица с прогрессом, статусом активности, фильтрация и сортировка.
- **Прогресс и статистика** – график успеваемости, таймлайн активности, алерты по упражнениям.
- **Отчёты** – экспорт списков в CSV, генерация PDF/DOCX для группы или отдельного студента, отправка на e‑mail (демо-режим).
- **Создание уроков** – форма с полями: название, язык/уровень, видео, текстовый материал, тестовые задания.
- **Избранное** – возможность добавлять курсы в избранное (сохраняется в localStorage).
- **Напоминания** – настраиваемые уведомления о занятиях.
- **localStorage** – сохраняются выбранные фильтры, роль, избранное; доступен сброс.
- **Адаптивный интерфейс** – корректное отображение на десктопе, планшете и мобильных устройствах.

## 🛠️ Технологический стек

- **React** – компонентная архитектура (проектируется под React).
- **Tailwind CSS** – утилитарная вёрстка, быстрая адаптивность.
- **MUI (Material-UI)** – для сложных компонентов (таблицы, диалоги).
- **RESTful API** – CRUD-маршруты для сущностей (планируется).
- **База данных (3НФ)** – 8+ таблиц: users, roles, courses, lessons, assignments, submissions, reminders, favorites, reports.
- **localStorage** – хранение пользовательских настроек в демо-режиме.

## 🎨 Дизайн-система

Проект следует единым правилам для обеспечения консистентности и удобства поддержки.

### 1. Единый подход
- Используем одну UI-систему на экран (Tailwind для вёрстки, MUI для сложных компонентов).
- Создаём переиспользуемые компоненты-обёртки (`AppButton`, `AppInput` и т.п.) для унификации стилей.

### 2. Дизайн-токены
- Все цвета, типографика, радиусы, тени и отступы задаются через токены (единые переменные), а не случайными значениями.

### 3. Цвета и контраст
- Семантические роли: `primary`, `success`, `warning`, `error`, `neutral`.
- Соблюдаем контрастность (WCAG AA). Цвет не является единственным носителем смысла — ошибки и успехи дублируем иконкой или текстом.

### 4. Типографика
- Единая шкала размеров (`h1`–`h6`, `body`, `caption`).
- Не используем больше 2–3 насыщенностей шрифта на одном экране.

### 5. Сетка, отступы, размеры
- Базовый шаг отступов фиксирован (кратный 4 px).
- Интерактивные элементы имеют зону клика не менее 44×44 px.

### 6. Компоненты и состояния
- Каждый интерактивный компонент поддерживает состояния: `hover`, `focus`, `active`, `disabled`, `loading`.
- Для блоков с данными предусмотрены состояния: `loading`, `empty`, `error`, `success`.

### 7. Адаптивность
- Mobile‑first подход. Брейкпоинты едины для всего проекта.
- На типовых разрешениях отсутствует горизонтальный скролл.

### 8. Доступность
- Все поля имеют явные метки (`label`), ошибки связаны с полем.
- Интерактивные элементы доступны с клавиатуры и имеют видимый фокус.

### 9. Контроль качества
- Перед релизом проверяем консистентность токенов, адаптивность, клавиатурную навигацию и контраст.
- Изменения дизайн-системы фиксируются в этом разделе README.

## 🏗️ Архитектура (планируемая)

- **Frontend**: React + TypeScript, маршрутизация (React Router), состояние (Context / Redux Toolkit).
- **Backend**: Node.js (Express) или Django (REST Framework) – на выбор команды.
- **API**: CRUD для пользователей, курсов, уроков, заданий, отчётов.
- **База данных**: PostgreSQL / MySQL с нормализацией (3НФ).

## 🧪 Запуск проекта (демо)

Для локального просмотра статического прототипа достаточно открыть файл `index.html` в браузере.  
Для разработки с React:

```bash
git clone https://github.com/StrAnnIk60RUS/LaboratoryWork.git
cd LaboratoryWork
npm install
npm run dev
```
