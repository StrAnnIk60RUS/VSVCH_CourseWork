# 09. Known pitfalls (anti-random)

Каталог типовых проблем для снижения случайных отклонений и повторяющихся ошибок.
Использовать как чеклист перед merge и при разборе инцидентов.

## Формат записи

- `Problem`: краткое название проблемы.
- `Symptoms`: как проявляется.
- `RootCause`: основная причина.
- `HowToCheck`: как быстро проверить.
- `Fix`: рекомендуемое исправление.
- `PreventiveRule`: какое правило не даст проблеме повториться.

---

## PIT-001: Рассинхрон API и страниц
- Problem: Изменили endpoint, но не обновили UI-документацию (или наоборот).
- Symptoms: Несоответствие маршрутов, payload или поведения на экранах.
- RootCause: Отсутствие doc-sync при изменении API/страниц.
- HowToCheck:
  - Сверить изменения с `docs/FUNCTIONAL_REQUIREMENTS.md` и `docs/PAGES_AND_FEATURES.md`.
  - Проверить, что затронутые FR-ID отражены в задаче.
- Fix:
  - Обновить оба документа в рамках той же задачи.
- PreventiveRule:
  - Включить обязательный doc-sync gate (`10-quality-gates.md`).

## PIT-002: Неполные role checks
- Problem: Логика доступа проверяется только частично (например, только auth без роли).
- Symptoms: Студент видит/выполняет teacher-операции или наоборот.
- RootCause: Пропуск middleware/условий роли в endpoint или UI-guard.
- HowToCheck:
  - Прогнать сценарии `student` и `teacher` для затронутых операций.
  - Убедиться, что недопустимые действия дают `403`.
- Fix:
  - Добавить/исправить role middleware и клиентские guards.
- PreventiveRule:
  - Любой ролевой сценарий проверять минимум для двух ролей.

## PIT-003: Неправильная работа localStorage
- Problem: Ключи UI-состояния несогласованы или сбрасываются не полностью.
- Symptoms: Фильтры/сортировки "залипают", кнопка сброса работает частично.
- RootCause: Разные префиксы/нейминг ключей, нет единой политики очистки.
- HowToCheck:
  - Проверить ключи на общий префикс (например, `vsvh:`).
  - Проверить сценарий "Сбросить настройки".
- Fix:
  - Унифицировать нейминг ключей и очистку по префиксу.
- PreventiveRule:
  - Следовать localStorage policy из мастер-плана.

## PIT-004: Незапрошенный рефактор
- Problem: В рамках задачи выполняются побочные рефакторы без согласования.
- Symptoms: Большой diff, изменения вне цели, рост риска регрессий.
- RootCause: Отсутствие ограничения scope/change budget.
- HowToCheck:
  - Сопоставить измененные файлы с `InScope` и `ChangeBudget`.
- Fix:
  - Вынести лишние изменения в отдельную задачу.
- PreventiveRule:
  - Запрет "silent refactor" в `MainRule.md`.

## PIT-005: Oversized change set
- Problem: Слишком много модулей в одном change.
- Symptoms: Сложно ревьюить, высокая вероятность скрытых побочных эффектов.
- RootCause: Нет лимитов на размер изменений.
- HowToCheck:
  - Сверить число файлов/модулей с `ChangeBudget` задачи.
- Fix:
  - Разбить задачу на последовательные инкременты.
- PreventiveRule:
  - Обязательный `ChangeBudget` и escalation при выходе за лимит.
