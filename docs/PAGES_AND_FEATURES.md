# Описание страниц и функционала — VSVH Languages

Документ фиксирует карту экранов веб-приложения, доступ по ролям и основные пользовательские функции. Детальные критерии приёмки API и бизнес-правила — в [`FUNCTIONAL_REQUIREMENTS.md`](./FUNCTIONAL_REQUIREMENTS.md).

---

## 0. Процесс применения документа

- Мастер-план проекта: `docs/preproject/03-architecture-and-api.md`.
- Обязательный workflow выполнения задач: `docs/preproject/04-execution-workflow.md`.
- Перед изменениями страниц сверять мастер-план (`§1.2`, `§17`, `§19`) и не смешивать целевое ТЗ с фактической готовностью.
- Перед закрытием задачи проходить DoD-gate (`§16`), перед финальной сдачей - `§20`.
- При изменении маршрутов/сценариев этот документ обновлять синхронно с `docs/FUNCTIONAL_REQUIREMENTS.md`.

## 1. Роли и доступ

| Роль | Обозначение | Принципиально отличается тем, что… |
|------|-------------|-------------------------------------|
| Гость | не авторизован | только просмотр публичного каталога и карточек курсов; нет записи, прогресса, кабинета. |
| Студент | `student` | запись на курсы, уроки, упражнения, избранное, напоминания, отчёт о своём прогрессе. |
| Преподаватель | `teacher` | CRUD своих курсов, уроков, упражнений; аналитика студентов; экспорт CSV; отчёт-сводка по курсу. |

Один пользователь может иметь обе роли (как в модели many-to-many с ролями); навигация показывает разделы, доступные хотя бы одной из ролей.

## 2. Страницы аутентификации (не входят в минимум «7 страниц» курсовой)

### 2.1 Регистрация

| Поле | Значение |
|------|----------|
| Маршрут | `/register` |
| Доступ | гость |
| Назначение | Создание учётной записи: e-mail, пароль, имя; выбор роли студент или преподаватель при регистрации. |
| Функции | Валидация полей; отображение ошибок API (409 «уже существует», 400); редирект после успеха (каталог или кабинет). |
| API | `FR-AUTH-01` |

### 2.2 Вход

| Поле | Значение |
|------|----------|
| Маршрут | `/login` |
| Доступ | гость |
| Назначение | Аутентификация по e-mail и паролю; сохранение JWT (например, в памяти + при необходимости refresh-стратегия по решению команды). |
| Функции | Единое сообщение при неверных данных; редирект для уже вошедшего пользователя. |
| API | `FR-AUTH-02` |

## 3. Основные страницы приложения (≥ 7, без учёта входа и регистрации)

Ниже перечислены **девять** экранов — с запасом относительно требования «не менее 7».

### 3.1 Главная (лендинг)

| Поле | Значение |
|------|----------|
| Маршрут | `/` |
| Доступ | все |
| Назначение | Краткое описание платформы, преимущества, призывы перейти в каталог или войти. |
| Функции | Навигация в каталог и страницы входа/регистрации; при авторизации — быстрые ссылки в кабинет студента или преподавателя. |
| Примечание | Семантическая разметка (`header`, `main`, `section`, `footer`). |

### 3.2 Каталог курсов

| Поле | Значение |
|------|----------|
| Маршрут | `/courses` |
| Доступ | все |
| Назначение | Список курсов с данными преподавателя и числом уроков. |
| Функции | **Фильтрация:** язык, уровень, минимальный рейтинг, поиск по названию/описанию. **Сортировка:** по дате создания, популярности (число записей в `enrollments` по курсу), рейтингу; порядок asc/desc. **Пагинация** с ограничением `limit` (на стороне API — максимум 50). Для студента: запись на курс, отписка (на карточке или на странице курса), добавление/удаление из избранного. |
| API | `FR-COURSE-01`, `FR-ENR-01`, `FR-ENR-03`, `FR-FAV-02`, `FR-FAV-03` |
| localStorage | Сохранение выбранных фильтров, параметров сортировки и номера страницы; кнопка сброса настроек (глобально — см. раздел 6). |

### 3.3 Карточка курса

| Поле | Значение |
|------|----------|
| Маршрут | `/courses/:courseId` |
| Доступ | все (просмотр); действия записи/избранного — авторизованный студент |
| Назначение | Полное описание курса, преподаватель, упорядоченный список уроков с числом упражнений. |
| Функции | Переход к уроку; для студента — запись/отписка, избранное; для пользователя из `course_staff` с правом ведения курса — ссылка в кабинет управления (`/teacher/courses/:courseId`). |
| API | `FR-COURSE-02`, `FR-LESSON-01`, записи и избранное — как в п. 3.2 |

### 3.4 Страница урока

| Поле | Значение |
|------|----------|
| Маршрут | `/courses/:courseId/lessons/:lessonId` |
| Доступ | просмотр контента — все; завершение урока и отправка ответов — студент с активной записью на курс |
| Назначение | Отображение текста урока, опционального видео; список упражнений с полем ответа и автопроверкой. |
| Функции | **Отметка урока как пройденного** (идемпотентно). **Отправка ответа** на упражнение с отображением результата (верно/неверно, балл). Обновление отображаемого прогресса по курсу после действий. |
| API | `FR-ENR-04`, `FR-SUB-01`, чтение уроков/упражнений — `FR-LESSON-01`, `FR-EX-01` |

### 3.5 Моё обучение (кабинет студента)

| Поле | Значение |
|------|----------|
| Маршрут | `/me/learning` |
| Доступ | студент (и пользователь с ролью student) |
| Назначение | Список записей на курсы с прогрессом %, ссылками на курс и уроки. |
| Функции | Быстрый переход к курсу; отписка от курса; опционально блок «избранное» со ссылкой на полный список (если избранное вынесено отдельно — см. п. 3.6). |
| API | `FR-ENR-02`, `FR-ENR-03`, `FR-FAV-01` |

### 3.6 Избранные курсы (опционально отдельным маршрутом)

| Поле | Значение |
|------|----------|
| Маршрут | `/me/favorites` |
| Доступ | авторизованный пользователь |
| Назначение | Список курсов из избранного; удаление из списка. |
| Функции | Карточки курсов с переходом на `/courses/:id`; синхронизация с API. |
| API | `FR-FAV-01`, `FR-FAV-03` |

*Примечание:* если избранное полностью реализовано только в каталоге и на карточке курса, отдельную страницу можно не выделять; тогда для подсчёта «7 страниц» достаточно усилить другой экран (например, объединить избранное с «Моё обучение» вкладками). В текущей карте **п. 3.6** даёт явный девятый сценарий навигации.

### 3.7 Прогресс и статистика

| Поле | Значение |
|------|----------|
| Маршрут | `/me/progress` |
| Доступ | студент |
| Назначение | Визуализация успеваемости: **график** (например, прогресс по курсам или динамика баллов), краткая сводка; **таймлайн** или список последних отправок (опора на историю submissions). |
| Функции | **Формирование отчёта о прогрессе** в PDF и DOCX; **запрос отправки отчёта на e-mail** (с учётом демо-режима без SMTP — `FR-RPT-05`). |
| API | `FR-SUB-02`, `FR-RPT-01`, `FR-RPT-02`, `FR-RPT-05` (type `student-progress`) |

### 3.8 Напоминания

| Поле | Значение |
|------|----------|
| Маршрут | `/me/reminders` |
| Доступ | авторизованный пользователь (в ТЗ избранное и напоминания привязаны к учётной записи; логично для студента) |
| Назначение | Список напоминаний, отсортированный по дате срабатывания. |
| Функции | **Создание**, **редактирование**, **удаление** напоминания; в UI — **снекбар** или тост при сохранении/ошибке; опционально визуальное выделение просроченных. |
| API | `FR-REM-01` — `FR-REM-03` |

### 3.9 Кабинет преподавателя — список курсов

| Поле | Значение |
|------|----------|
| Маршрут | `/teacher/courses` |
| Доступ | преподаватель |
| Назначение | Курсы, где пользователь — владелец; счётчики уроков и записей. |
| Функции | Переход к управлению курсом; **создание нового курса** (отдельный маршрут `/teacher/courses/new` или модальное окно — на усмотрение реализации). |
| API | `FR-TCH-01`, `FR-COURSE-03` |

### 3.10 Управление курсом (преподаватель)

| Поле | Значение |
|------|----------|
| Маршрут | `/teacher/courses/:courseId` |
| Доступ | преподаватель — только владелец курса |
| Назначение | Редактирование метаданных курса; **CRUD уроков** и **CRUD упражнений**; **таблица студентов** с фильтром по активности (`active` / `inactive`), сортировкой по имени, прогрессу, активности; **скачивание CSV** списка студентов; **отчёт PDF/DOCX** по сводке курса; отправка отчёта на e-mail. |
| Функции | Удаление курса с подтверждением (диалог). Экспорт и отчёты — явные кнопки с индикацией загрузки. |
| API | `FR-COURSE-04`, `FR-COURSE-05`, `FR-LESSON-02`, `FR-LESSON-03`, `FR-EX-02`, `FR-TCH-02`, `FR-TCH-03`, `FR-RPT-03`, `FR-RPT-04`, `FR-RPT-05` (type `course-summary`) |

### 3.11 Профиль и настройки

| Поле | Значение |
|------|----------|
| Маршрут | `/me/profile` или `/settings` |
| Доступ | авторизованный пользователь |
| Назначение | Отображение и **обновление имени** (`name`); выход из системы. |
| Функции | **Кнопка «Сбросить настройки интерфейса»** — полная очистка ключей приложения в `localStorage` (фильтры, сортировка, пагинация, прочие сохранённые предпочтения UI). |
| API | `FR-AUTH-03` (текущий пользователь), `FR-USER-01`, `FR-USER-02` |

## 5. Соответствие требованиям курсовой (чек-лист)

| Требование | Как закрывается в документе |
|------------|-----------------------------|
| ≥ 2 роли с разным функционалом | Студент и преподаватель; отдельные маршруты `/me/*` и `/teacher/*`. |
| Страницы входа и регистрации | П. 2. |
| ≥ 7 страниц кроме auth | П. 3.1–3.11 (лендинг, каталог, курс, урок, обучение, избранное, прогресс, напоминания, список курсов преподавателя, управление курсом, профиль). |
| ≥ 15 функций (не auth) | Фильтры/сортировка/пагинация каталога, запись/отписка, избранное, завершение урока, отправка ответа, CRUD курса/уроков/упражнений, фильтр/сортировка таблицы студентов, CSV, напоминания CRUD, график/таймлайн, 2 типа отчётов + e-mail, обновление имени, сброс localStorage. |
| ≥ 2 отчёта (PDF/DOCX/e-mail) | Отчёт студента (`FR-RPT-01/02/05`), сводка по курсу (`FR-RPT-03/04/05`). |
| localStorage + сброс | П. 3.2, 3.11. |
| REST, CRUD, 14 таблиц 3НФ (целевая схема) | См. `docs/DATABASE.md`, `FUNCTIONAL_REQUIREMENTS.md` §5, README. |

## 6. Рекомендуемые ключи localStorage (пример)

Имена согласовать в коде единообразно; при сбросе удалять все префиксом приложения, например `vsvh:`.

| Ключ (пример) | Содержимое |
|---------------|------------|
| `vsvh:catalogFilters` | язык, уровень, minRating, search |
| `vsvh:catalogSort` | поле сортировки, порядок |
| `vsvh:catalogPage` | номер страницы |
| `vsvh:teacherStudentsTable` | последний фильтр/сортировка для таблицы (если сохраняется на клиенте) |

Кнопка сброса на странице профиля/настроек удаляет перечисленные ключи (или весь префикс `vsvh:`).
